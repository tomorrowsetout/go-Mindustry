# 双核心架构集成指南

## 概述

mdt-server 已经实现了双核心架构，将游戏逻辑和 IO 操作分离到两个核心：

- **Core1 (Game Loop)**: 运行在主线程，处理 60 TPS 的实时游戏逻辑
- **Core2 (IO Core)**: 运行在独立 goroutine，处理所有 IO 密集型任务

## 核心组件

### Core1 (Game Loop)
- 运行在单个核心上
- 处理 60 TPS 的实时游戏逻辑
- 调用 `wld.Step()` 更新世界状态
- 处理实体同步、波次更新等

### Core2 (IO Core)
- 4 个 worker goroutine
- 处理网络包收发
- 处理存档保存/加载
- 处理 Mod 加载/卸载
- 处理事件记录
- 处理 MSAV 文件读写

## 集成步骤

### 1. 创建 ServerCore

```go
import (
    "time"
    "mdt-server/internal/core"
    "mdt-server/internal/config"
)

// 创建 IO Core 配置
ioConfig := core.Config{
    Name:        "io-core",
    MessageBuf:  30000,
    WorkerCount: 4,
}

// 创建持久化配置
persistCfg := config.PersistConfig{
    Enabled:     true,
    Directory:   "data/state",
    File:        "server-state.json",
    IntervalSec: 30,
    SaveMSAV:    true,
    MSAVDir:     "data/snapshots",
}

// 创建 ServerCore
serverCore := core.NewServerCore(
    16666666 * time.Nanosecond, // 60 TPS
    ioConfig,
    persistCfg,
)
```

### 2. 设置 Game Loop 回调

```go
serverCore.SetGameTickFn(func(tick uint64, delta time.Duration) {
    // 执行游戏逻辑
    wld.Step(delta)
    
    // 广播实体同步
    // 更新波次
    // 处理玩家更新
})
```

### 3. 启动核心

```go
// 启动所有核心
serverCore.StartAll()

// Core1 由主线程调用 Run() 启动
gameInterval := 16666666 * time.Nanosecond // 60 TPS
serverCore.Core1.Run(gameInterval)
```

### 4. 发送消息到 Core2

```go
// 保存状态
serverCore.SendToCore2(&core.PersistenceMessage{
    Action: "save_state",
    Path:   statePath,
})

// 加载世界模型
serverCore.SendToCore2(&core.WorldStreamMessage{
    Action: "load_model",
    Path:   "map.msav",
})

// 记录事件
serverCore.SendToCore2(&core.StorageMessage{
    Action:    "record_event",
    EventData: eventData,
})
```

## 消息类型

### Core1 消息 (Game Loop)
- `GameTickMessage` - 世界 tick
- `WaveUpdateMessage` - 波次更新
- `EntityBroadcastMessage` - 实体同步广播
- `PlayerUpdateMessage` - 玩家更新
- `BuildQueueProcessMessage` - 建筑队列处理

### Core2 消息 (IO Core)
- `PacketMessage` - 网络包收发
- `ConnectionMessage` - 连接事件
- `PersistenceMessage` - 存档操作
- `ModMessage` - Mod 操作
- `StorageMessage` - 事件记录
- `WorldStreamMessage` - MSAV 文件读写

## 完整示例

```go
package main

import (
    "time"
    "mdt-server/internal/core"
    "mdt-server/internal/config"
    "mdt-server/internal/world"
)

func main() {
    // 创建配置
    ioConfig := core.Config{
        Name:        "io-core",
        MessageBuf:  30000,
        WorkerCount: 4,
    }
    
    persistCfg := config.PersistConfig{
        Enabled:     true,
        Directory:   "data/state",
        File:        "server-state.json",
        IntervalSec: 30,
    }
    
    // 创建 ServerCore
    serverCore := core.NewServerCore(
        16666666*time.Nanosecond, // 60 TPS
        ioConfig,
        persistCfg,
    )
    
    // 创建世界
    wld := world.New(world.Config{TPS: 60})
    
    // 设置 Game Loop 回调
    serverCore.SetGameTickFn(func(tick uint64, delta time.Duration) {
        wld.Step(delta)
        
        // 广播实体同步
        entities := wld.DrainEntityEvents()
        // ... 处理实体事件
    })
    
    // 启动 Core2
    serverCore.StartAll()
    
    // 启动 Core1 (主线程)
    gameInterval := 16666666 * time.Nanosecond
    serverCore.Core1.Run(gameInterval)
}
```

## 迁移现有代码

### 从单线程到双核心

1. **识别 IO 操作**: 找出所有 IO 操作（文件读写、网络收发、数据库操作）
2. **创建消息**: 将 IO 操作封装为消息发送到 Core2
3. **移除同步调用**: 移除同步的 IO 调用，改为异步消息
4. **处理回调**: 使用 ResultChan 处理异步结果

### 示例：存档操作

**单线程版本**:
```go
// 同步保存
err := persist.Save(cfg.Persist, state)
```

**双核心版本**:
```go
// 异步发送消息
resultChan := make(chan core.PersistenceResult, 1)
serverCore.SendToCore2(&core.PersistenceMessage{
    Action:     "save_state",
    Path:       statePath,
    ResultChan: resultChan,
})

// 接收结果
result := <-resultChan
if result.Error != nil {
    // 处理错误
}
```

## 性能优化建议

1. **减少消息频率**: 批量发送消息，避免频繁通信
2. **使用缓冲通道**: Core2 的消息缓冲区设置为 30000
3. **异步处理**: 尽量使用异步消息，避免阻塞 Game Loop
4. **监控队列**: 定期检查 Core2 队列大小，避免积压

## 故障排查

### Core2 队列积压
```go
running, stats := serverCore.Stats()
if stats[3] > 10000 {
    log.Warn("Core2 queue is too large", "size", stats[3])
}
```

### 消息丢失
```go
if !serverCore.SendToCore2(msg) {
    log.Warn("Failed to send message to Core2")
}
```
